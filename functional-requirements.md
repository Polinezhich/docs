# Функциональные требования — VetHub MVP

> Описание охватывает ключевые use case всех акторов платформы в рамках MVP.

## Акторы платформы

| Актор | Роль |
|---|---|
| Владелец питомца | Конечный пользователь: поиск, запись, оплата |
| Фриланс-ветеринар | Самозанятый специалист: расписание, подтверждение записей |
| Администратор клиники | Управление профилем клиники, врачами, слотами |
| Администратор VetHub | Модерация, контроль платформы |

---

## UC-001 — Авторизация владельца питомца

| Элемент | Описание |
|---|---|
| **Акторы** | Владелец питомца |
| **Предусловия** | Пользователь установил приложение VetHub |
| **Постусловия** | Пользователь авторизован, доступны все функции MVP |

**Основной поток:**
1. Пользователь открывает приложение
2. Система показывает экран авторизации
3. Пользователь выбирает: «Войти» / «Регистрация»
4a. Вход: вводит телефон → SMS-код → профиль загружен
4b. Регистрация: вводит телефон → SMS-код → ФИО → профиль создан
5. Система создаёт JWT-токен
6. Пользователь попадает на главный экран поиска

**Альтернативные потоки:**
- `3a` — Вход по Face ID / Touch ID (сохранённые данные)
- `3b` — Регистрация через Apple/Google → Автозаполнение

**Исключительные потоки:**
- `4c` — Неверный SMS-код (3 попытки) → «Повторить код через 60 сек»
- `4d` — Телефон не российский → «Поддерживаются только номера РФ»

**Бизнес-правила:**
- Единый номер на 1 аккаунт
- JWT-токен 7 дней, refresh автоматически
- Обязательные поля: телефон, ФИО

## UC-002 — Поиск ветеринарных клиник

| Элемент | Описание |
|---|---|
| **Акторы** | Владелец питомца |
| **Предусловия** | Пользователь авторизован, геолокация включена |
| **Постусловия** | Отображён актуальный список клиник в радиусе 5 км |

**Основной поток:**
1. Пользователь открывает экран поиска
2. Система определяет текущую геопозицию
3. Пользователь выбирает фильтры (специализация, рейтинг, расстояние)
4. Система отображает 10 ближайших клиник с рейтингом, расстоянием, свободными слотами
5. Пользователь просматривает детали клиники (врачи, цены, отзывы)

**Альтернативные потоки:**
- `2a` — Геолокация отключена → предложить ввести адрес вручную
- `3a` — Нет фильтров → показать все клиники по расстоянию

**Исключительные потоки:**
- `4a` — Нет клиник поблизости → «Расширить радиус поиска?»

**Бизнес-правила:**
- Максимум 50 клиник на экране
- Рейтинг считается по последним 100 отзывам

## UC-003 — Онлайн-запись к ветеринару

| Элемент | Описание |
|---|---|
| **Акторы** | Владелец питомца |
| **Предусловия** | Клиника выбрана из поиска (UC-002), профиль питомца создан (UC-004) |
| **Постусловия** | Запись создана со статусом «Подтверждено», отправлено push-уведомление |

**Основной поток:**
1. Пользователь нажимает «Записаться»
2. Система показывает доступные слоты на 7 дней вперёд
3. Пользователь выбирает дату, время, врача
4. Система отображает стоимость услуги
5. Пользователь оплачивает картой
6. Клиника получает уведомление

**Альтернативные потоки:**
- `5a` — Отказ от оплаты → «Запись сохранена как черновик»
- `6a` — Клиника отклонила → предложить альтернативные слоты

**Исключительные потоки:**
- `2a` — Нет свободных слотов → «Уведомить о появлении мест?»

## UC-004 — Управление профилем питомца

| Элемент | Описание |
|---|---|
| **Акторы** | Владелец питомца |
| **Предусловия** | Пользователь авторизован |
| **Постусловия** | Профиль сохранён и доступен при создании записи |

**Основной поток:**
1. Пользователь заходит в «Профиль питомца»
2. Заполняет данные: порода, возраст, вес, прививки
3. Загружает фото питомца
4. Сохраняет профиль
5. Система присваивает уникальный ID питомца

**Альтернативные потоки:**
- `2a` — Редактирование существующего профиля

**Исключительные потоки:**
- `2b` — Некорректные данные → показать ошибки валидации

**Бизнес-правила:**
- Вес в диапазоне 0.1–200 кг
- Возраст 0–30 лет

## UC-005 — Регистрация профиля ветеринара

| Элемент | Описание |
|---|---|
| **Акторы** | Ветеринар-фрилансер |
| **Предусловия** | Ветеринар имеет диплом, сертификаты, ИНН |
| **Постусловия** | Профиль ветеринара доступен для записи в поиске |

**Основной поток:**
1. Ветеринар переходит по ссылке «Стать ветеринаром»
2. Заполняет профиль: ФИО, специальности, опыт, фото
3. Загружает сканы: диплом, сертификаты, лицензия
4. Указывает график: дни/время приёма, цены услуг
5. Администратор VetHub модерирует (24 ч)
6. Профиль активирован, ветеринар получает уведомление

**Альтернативные потоки:**
- `5a` — Модерация отклонена → указать причину + повторная подача

**Исключительные потоки:**
- `3a` — Документы нечитаемы → запрос на перезагрузку

**Бизнес-правила:**
- Минимум 3 года опыта
- 2+ специальности
- Рейтинг >4.5 для попадания в топ

## UC-006 — Регистрация клиники в агрегаторе

| Элемент | Описание |
|---|---|
| **Акторы** | Администратор клиники |
| **Предусловия** | Клиника имеет лицензию Россельхознадзора, адрес, контакты |
| **Постусловия** | Клиника отображается в поиске с актуальными слотами |

**Основной поток:**
1. Администратор переходит по ссылке «Подключить клинику»
2. Заполняет профиль: название, адрес, график работы
3. Добавляет врачей (ФИО, специальности, сертификаты)
4. Загружает услуги с ценами
5. Загружает фото клиники и оборудования
6. Администратор VetHub модерирует (48 ч)
7. Клиника активирована

**Исключительные потоки:**
- `4a` — Цены >среднерыночных в 2 раза → запрос обоснования

**Бизнес-правила:**
- Минимум 2 врача
- Минимум 12 ч работы в день
- Цены ±30% от среднерыночных

## UC-007 — Подтверждение / отмена записи

| Элемент | Описание |
|---|---|
| **Акторы** | Ветеринар-фрилансер / Администратор клиники |
| **Предусловия** | Поступила новая запись от владельца (UC-003) |
| **Постусловия** | Статус записи обновлён для всех сторон |

**Основной поток:**
1. Ветеринар/администратор получает push/email о новой записи
2. Открывает детали: питомец, услуга, время
3. Подтверждает («Приму») или отменяет («Перенести»)
4. Владелец получает уведомление
5. Статус записи меняется на «Подтверждено» / «Отменено»

**Альтернативные потоки:**
- `3a` — Перенос → предложить 3 альтернативных слота

**Исключительные потоки:**
- `4a` — Нет ответа за 15 мин → автоподтверждение

**Бизнес-правила:**
- Отмена менее чем за 2 ч до приёма = штраф 20% от стоимости

## UC-008 — Оплата ветеринарных услуг картой

| Элемент | Описание |
|---|---|
| **Акторы** | Владелец питомца |
| **Предусловия** | Запись создана (UC-003), профиль питомца заполнен (UC-004) |
| **Постусловия** | Платёж завершён, средства списаны, ветеринар уведомлён, чек сохранён |

**Основной поток:**
1. Пользователь нажимает «Оплатить» на экране записи
2. Система отображает сумму, детали записи, способы оплаты
3. Пользователь выбирает «Карта» → открывается форма Тинькофф.Касса
4. Вводит данные карты (номер, срок, CVC)
5. Подтверждает 3-D Secure кодом из SMS
6. Система проводит платёж, возвращает статус «Успешно»
7. Запись меняет статус на «Оплачено и подтверждено»

**Альтернативные потоки:**
- `4a` — Сохранение карты → «Карта сохранена для будущих оплат»
- `6a` — 3-D Secure по биометрии → автоматическое подтверждение

**Исключительные потоки:**
- `4b` — Неверные данные карты → «Проверьте номер/срок/CVC»
- `5b` — Недостаточно средств → «Пополните карту и оплатите снова»
- `5c` — 3-D Secure таймаут → «Код не введён за 2 минуты. Повторите оплату»

**Бизнес-правила:**
- Предоплата 100% стоимости услуги
- Поддержка Mir / VISA / Mastercard / Mir Pay
- Комиссия платформы 15% (не видна пользователю)
- Срок действия карты >30 дней
- Авторекуррент для повторных записей
