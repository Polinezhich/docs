# Стратегия платформизации VetHub

## Почему VetHub — платформа?

VetHub платформизуется, так как его core-активность — это **мэтчинг спроса и предложения** на рынке ветеринарных услуг. Корпоративные клиенты могут переиспользовать компоненты платформы.

---

## B2B-сегменты

| Сегмент | Пример компаний | Что купят | Ценность |
|---|---|---|---|
| Сети ветклиник | «Белый Клык», «МедВет» | Запись по API (встраивание в их сайт/приложение) | Синхронизация расписаний |
| Крупные работодатели / ЖК | Сбер, Яндекс, «Самолёт» | Коробочное B2B2E-решение (для сотрудников и жителей) | Нематериальный бенефит |
| Платформы для зообизнеса | Doggy, Petstory | API поиска ближайшей ветклиники с брендированием | Расширение функционала |

---

## Бизнес-модель: B2B2C

- **B (VetHub):** предлагает технологические решения (API, виджеты)
- **B (партнёр):** корпоративные клиенты покупают доступ и встраивают в свои сервисы
- **C (пользователь):** владельцы питомцев взаимодействуют через интерфейсы партнёров или напрямую через VetHub

## Стратегия монетизации

| Механизм | Применение | Доля выручки |
|---|---|---|
| Комиссия 15% | С каждой записи через платформу | 60% |
| B2B API-подписка | Фиксированный платёж + тарификация по числу вызовов | 25% |
| Премиум B2C | 299 руб/мес: приоритетная запись, скидка 10%, топ-врачи | 10% |
| Продажа доли инвесторам | Серии A/B для масштабирования в регионы | — |

## Техническая реализация

Переход от монолита к модульному монолиту, затем к управляемым микросервисам:

**1. Выделяемые модули:**
- Авторизация и профили питомцев
- Алгоритм подбора клиник, кэш расстояний
- Транзакции записи + оплата

**2. API-шлюз** — единая точка входа для внешних партнёров:
- Аутентификация по API-ключам (B2B)
- Версионирование (старые интеграции не ломаются)
- Преобразование внутренних форматов в публичный OpenAPI spec

**3. Кэш поиска (Redis):**
- Список клиник в радиусе 1 км — TTL 10 минут
- Инвалидация при изменении расписания клиники или отзывов
- Отдельный инстанс Redis для независимого масштабирования

**4. White-label виджет:**
- Партнёр встраивает виджет записи в свой сайт
- Партнёр может менять цвета и логотип
- Ядро (логика, данные) остаётся за VetHub
- Старый монолит не трогается — новый платформенный слой постепенно забирает нагрузку

## Roadmap платформизации

| Этап | Сроки | Действия |
|---|---|---|
| MVP | 1–6 мес | Базовый поиск и запись. API для партнёров не предоставляется. Проверка PMF. |
| Пилот B2B2C | 9–12 мес | API для 2–3 пилотных партнёров (клиники). White-label виджет на сайтах партнёров. |
| Масштабирование платформы | 12–18 мес | 20+ B2B-клиентов, SLA, лимиты, документация. Переход к модульному монолиту и микросервисам. |
| Экосистема | 18+ мес | Интеграция страхования, зооаптек. Открытый API для сторонних разработчиков. Опция продажи бигтеху. |
