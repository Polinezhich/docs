## Непокрытые категории и план развития

### Ёмкость
- **Почему не критична для MVP:** объём данных минимален (~1 МБ/профиль питомца). PostgreSQL легко масштабируется до 10 ГБ.
- **Риски игнорирования:** нет, дефолтные лимиты облака покрывают MVP.
- **План развития:** v2.0 — лимит 50 ТБ хранилища.

### Наблюдаемость
- **Почему не критична:** простая инфраструктура (1–2 сервера), нет сложных микросервисов.
- **Риски игнорирования:** затруднённая диагностика при >50k MAU.
- **План развития:** v1.5 — Яндекс.Мониторинг, Sentry для логов.

### Удобство использования
- **Почему не критична:** базовый UX по шаблонам (Яндекс.Карты, DocDoc) достаточен для PMF-валидации.
- **Риски игнорирования:** NPS < 50, отток пользователей из-за интерфейса.
- **План развития:** v1.5 — A/B-тесты UI, тепловые карты Hotjar.

### Поддерживаемость
- **Почему не критична:** компактная внутренняя команда (8 человек), монолитная архитектура.
- **Риски игнорирования:** затруднения при найме новых разработчиков.
- **План развития:** v2.0 — Swagger API, 80% тест-покрытие.
